## Порядок выполнения (чеклист)

- [ ] 1. Создать ПланВидовХарактеристик (ПВХ) `ДополнительныеСвойства`, отнести к `Общее → Справочники`.
- [ ] 2. Создать справочник `ЗначенияСвойств` (`Общее → Справочники`), подчинить созданному ПВХ.
- [ ] 3. В ПВХ установить тип значения характеристики: ссылка на `ЗначенияСвойств`; в «Дополнительные значения характеристик» указать `ЗначенияСвойств`.
- [ ] 4. Создать предопределенный элемент ПВХ `материал` (тип — как в методичке).
- [ ] 5. Создать регистр сведений `СвойстваНоменклатуры` (`Общее → Журналы`): измерения `Номенклатура`, `Свойство`; ресурс `ЗначениеСвойства`.
- [ ] 6. Создать регистр сведений `ЦеныПродажиНоменклатуры` (измерение `Номенклатура`, ресурс `Цена` 15:2, неотриц.), периодичность «в пределах дня», режим записи «подчинение регистратору», подсистема `Продажи → Журналы`.
- [ ] 7. Создать документ `УстановкаЦенНоменклатуры` (`Продажи → Документы`) с реквизитами `Номенклатура`, `Цена` (15:2, неотриц.).
- [ ] 8. У регистра `ЦеныПродажиНоменклатуры` задать регистратором документ `УстановкаЦенНоменклатуры`.
- [ ] 9. В документе `УстановкаЦенНоменклатуры` на вкладке «Движения» запустить конструктор движений → «Заполнить выражения» → ОК.
- [ ] 10. Открыть форму документа `ПродажаТоваров` (или аналог из прошлых лаб), заменить функцию `ПолучитьЦенуНоменклатуры` на вариант из методички (получение цены через `СрезПоследних` регистра).
- [ ] 11. Настроить функциональные опции (если требуется методичкой) под обновлённую конфигурацию.
- [ ] 12. Обновить конфигурацию БД, проверить установку/получение цен.
- [ ] 13. Выгрузить `.dt` (если требуют).
